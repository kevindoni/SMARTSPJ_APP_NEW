# Plan: Bug Audit Auto Update

## Findings

Setelah mengecek seluruh kode auto-update (main process, preload, dan frontend), berikut bug yang ditemukan:

---

### BUG 1 (Medium): Race condition auto-check vs manual check

**File**: `electron/main.js:326-340`

Auto-check berjalan 5 detik setelah `did-finish-load`. Jika user klik "Cek Update" sebelum/during auto-check, **dua** `checkForUpdates()` berjalan bersamaan. Keduanya trigger event listener yang sama (`update-available`, `update-not-available`, `update-error`), menyebabkan:

- State bisa flicker (misal: `available` → `not-available` jika `isNewerVersion` return berbeda dari electron-updater)
- Duplicate events ke renderer

**Fix**: Tambahkan flag `isChecking` untuk mencegah concurrent check. Suppress auto-check events jika user sedang manual check.

---

### BUG 2 (Low): `isNewerVersion` tidak handle pre-release tag

**File**: `electron/main.js:39-49`

```js
const r = remote.replace(/^v/, '').split('.').map(Number);
```

Jika versi mengandung pre-release seperti `1.0.0-beta`, segment `"0-beta"` → `NaN`. Semua perbandingan `NaN > x` dan `NaN < x` return `false`, sehingga function selalu return `false` (tidak ada update) untuk versi pre-release.

**Fix**: Strip non-numeric suffix dari setiap segment sebelum `Number()`:

```js
const r = remote
  .replace(/^v/, '')
  .split('.')
  .map((s) => parseInt(s, 10) || 0);
```

---

### BUG 3 (Low): Tidak ada debounce/lock pada check button

**File**: `src/components/layout/Header.jsx:80-101`

User bisa klik "Cek Update" berkali-kali dengan cepat, trigger banyak `checkForUpdates()` call secara concurrent. electron-updater bisa throw error jika ada check yang sedang berjalan.

**Fix**: Tambahkan guard `if (updateStatus === 'checking') return;` di awal `handleCheckUpdate`.

---

### BUG 4 (Low - UX): Auto-check mengubah state dari `idle` tanpa user action

**File**: `electron/main.js:326-340`

Auto-check 5 detik setelah load bisa mengubah status dari `idle` ke `not-available` atau `error`. User melihat button berubah dari "v1.0.0" ke "v1.0.0 ✓" atau "Update gagal" tanpa melakukan apapun.

**Ini sebenarnya by design** untuk auto-update UX, tapi jika ingin suppress: hanya trigger `update-available` event dari auto-check, bukan `update-not-available` atau `update-error`.

---

### NON-BUG (Security Note)

`data/updater.json` berisi GitHub token plain-text (`gho_...`). File ini sudah di `.gitignore` (`data/*.json`) jadi tidak ter-commit ke repo. Code sudah ada mekanisme migrasi ke encrypted storage (`safeStorage`). **Tidak ada masalah security di repo.**

---

## Recommended Fixes (Priority Order)

| #   | Bug                                 | Priority      | Effort  |
| --- | ----------------------------------- | ------------- | ------- |
| 1   | Race condition auto-check vs manual | Medium        | Small   |
| 2   | `isNewerVersion` NaN parsing        | Low           | Trivial |
| 3   | No debounce on check button         | Low           | Trivial |
| 4   | Auto-check state override           | Low/By Design | Small   |

## Implementation Plan

1. **Fix `isNewerVersion`** — ubah `.map(Number)` jadi `.map(s => parseInt(s, 10) || 0)` di `electron/main.js:40-41`
2. **Add concurrency guard** — tambah `let isCheckingUpdate = false` flag di `setupAutoUpdater()`, set true sebelum check, false setelah selesai
3. **Add debounce in frontend** — tambah early return di `handleCheckUpdate` jika status sudah `checking`
4. **Suppress auto-check non-critical events** (optional) — hanya emit `update-available` dari auto-check, bukan `update-not-available`/`update-error`
