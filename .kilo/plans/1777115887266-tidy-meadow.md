# Rencana Dokumentasi Pendukung untuk Sistem SmartSPJ

Berikut adalah daftar dokumen pelengkap atau lampiran yang direkomendasikan ditambahkan untuk meningkatkan kelengkapan sistem SmartSPJ, terutama bagi pengguna akhir, administrator, dan pengembang yang mungkin terlibat dalam pemeliharaan atau integrasi sistem.

## 1. Dokumen Untuk Pengguna Akhir (Bendahara Sekolah)

1. **Panduan Pengguna SmartSPJ (User Manual)**
   - Panduan langkah demi langkah mulai dari instalasi, aktivasi lisensi, hingga penggunaan masing-masing modul.
   - Termasuk screenshot UI, penjelasan field, dan contoh penggunaan.

2. **Panduan Instalasi dan Konfigurasi Awal**
   - Persyaratan sistem (OS, RAM, storage).
   - Langkah-langkah download installer, menjalankan wizard instalasi.
   - Konfigurasi koneksi ke database ARKAS (path, kredensial).
   - Aktivasi lisensi (online/offline).

3. **Panduan Pemecahan Masalah (Troubleshooting Guide) dan FAQ**
   - Masalah umum dan solusinya (misalnya: gagal koneksi ARKAS, lisensi tidak terdeteksi, aplikasi crash saat startup, masalah printing).
   - Daftar kode error dan penjelasannya.
   - Pertanyaan yang sering diajukan (FAQ) mengenai fitur, batasan trial, dan pembaruan.

4. **Panduan Backup dan Pemulihan Data**
   - Cara membuat backup lengkap dan backup hanya konfigurasi.
   - Langkah-langkah restore dari backup.
   - Jadwal backup yang disarankan dan lokasi penyimpanan yang aman.

5. **Panduan Cetak dan Ekspor Dokumen**
   - Cara mencetak kwitansi, laporan, dan bukti transaksi.
   - Pengaturan template, margin, dan watermark (untuk versi trial).
   - Ekspor ke format PDF, Excel, dan CSV.

6. **Glossarium Istilah dan Singkatan**
   - Definisi istilah akuntansi sekolah (BOS, SiLPA, RKAS, etc.) dan singkatan yang digunakan di aplikasi.
   - Penjelasan kolom kunci di tabel database jika diperlukan.

## 2. Dokumen Untuk Administrator / Teknis

7. **Panduan Administrasi Lisensi dan Pengaturan Sistem**
   - Cara memberikan lisensi ke pengguna baru (generate license key via admin tools).
   - Mengelola masa trial, memperpanjang lisensi, dan mentransfer lisensi antar PC.
   - Menggunakan fitur deaktivasi/aktivasi manual melalui menu Pengaturan.

8. **Panduan Keamanan dan Privasi Data**
   - Penjelasan enkripsi data (SQLCipher), penyimpanan lisensi di safeStorage, dan proteksi file-file sensitif.
   - Kebijakan penyimpanan log dan data pengguna.
   - Pedoman untuk menjaga kerahasiaan NPSN dan data keuangan sekolah.

9. **Panduan Integrasi dengan Sistem ARKAS dan Sistem Lain**
   - Spesifikasi format data yang diimpor/diexport (JSON, CSV).
   - Cara mengatur sinkronisasi otomatis atau manual dengan database ARKAS.
   - Contoh skrip atau query untuk mengekstrak data dari ARKAS ke format yang dapat dipakai SmartSPJ.

10. **Dokumen Struktur Database (Data Dictionary)**
    - Diagram ER (Entity-Relationship) tabel utama: sekolah, transaksi, anggaran, pengguna, lisensi, dsb.
    - Deskripsi setiap kolom: tipe data, constraint, contoh nilai, dan keterangan.
    - Catatan mengenai field yang dienkripsi atau yang merupakan hasil perhitungan.

11. **Release Notes dan Changelog Terperinci**
    - Daftar perubahan per versi (fitur baru, perbaikan bug, peningkatan performa) dalam format yang mudah dibaca.
    - Informasi tentang versi Electron, Node.js, dan dependensi utama yang digunakan.
    - Panduan upgrade dari versi lama ke versi baru (migrasi data, perubahan konfigurasi).

## 3. Dokumen Untuk Pengembang (Jika Ada Kontribusi Masuk)

12. **Panduan Pengembang SmartSPJ (Developer Guide)**
    - Struktur folder dan kode sumber (penjelasan masing-masing direktori: src, electron, scripts, docs).
    - Panduan setup environment development (Node, Yarn/npm, vite, electron-builder).
    - Cara menjalankan aplikasi dalam mode development (`npm run dev`).
    - Panduan build untuk produksi (`npm run build:electron`).
    - Coding conventions: gaya penulisan JavaScript/React, penggunaan Tailwind CSS, dan aturan commit.
    - Instruksi menjalankan unit test atau end-to-end test (jika ada).
    - Panduan menambahkan fitur baru: menambah IPC handler, menambahkan menu di Sidebar, dan menambahkan enforcer lisensi.

13. **API Dokumentasi Internal (Jika Diperlukan)**
    - Dokumentasi fungsi-fungsi utama di `electron/handlers/*` dan `src/services/*`.
    - Contoh pemanggilan lewat `ipcRenderer`/`ipcMain`.
    - Skema objek yang ditransfer antara proses utama dan renderer.

14. **Panduan Kontribusi dan Pengelolaan Repository**
    - Cara membuat branch, pull request, dan kode review.
    - Panduan pembuatan release otomatis (github release) menggunakan `gh` CLI.
    - Instruksi untuk menjalankan lint dan precommit hooks.

## 4. Dokumen Legal dan Kebijakan

15. **Kebijakan Privasi Data (Privacy Policy)**
    - Jenis data yang dikumpulkan (NPSN, nama sekolah, data transaksi, hardware fingerprint).
    - Tujuan penggunaan data (aktivasi lisensi, pemeliharaan aplikasi).
    - Bagaimana data disimpan dan siapa yang dapat mengaksesnya (hanya pengguna setempat, kecuali jika mengirim crash report secara opsional).
    - Hak pengguna untuk menghapus data atau membatasi pengumpulan.

16. **Perjanjian Layanan dan Pembayaran (Terms of Service)**
    - Ketika pengguna membeli lisensi Basic/Pro: metode pembayaran, politika pengembalian uang, dan pembaruan otomatis.
    - Ketentuan mengenai pembaruan harga dan fitur di masa depan.

17. **Surat Pernyataan Keamanan dan Kelayakan (Security Attestation)**
    - Ringkasan langkah-langkah yang telah dilakukan untuk mengamankan aplikasi (enkripsi, code obfuscation, license binding).
    - Informasi tentang penetasian keamanan yang telah dilakukan (jika ada).

## 5. Lampiran dan Referensi

18. **Lampiran A: Contoh Format File Backup dan Ekspor**
    - Struktur file JSON backup.
    - Contoh file Excel hasil ekspor BKU, laporan, dan register kas.

19. **Lampiran B: Contoh Laporan dan Tampilan Cetak**
    - Sampel output laporan keuangan, BA Rekonsiliasi, dan nota group manager.
    - Contoh kwitansi A2 dengan dan tanpa watermark.

20. **Lampiran C: Daftar Dependensi dan Versi**
    - Daftar lengkap paket npm (produksi dan development) dengan versi yang digunakan.
    - Informasi tentang lisensi setiap dependensi (untuk kepatuhan hukum).

21. **Lampiran D: Troubleshooting Log dan Cara Mengumpulkan Debug Info**
    - Cara mengaktifkan logging tingkat debug (`ELECTRON_ENABLE_LOGGING=1`).
    - Lokasi file log dan cara mengirimkannya ke tim support bila diperlukan.

---

### Prioritas Implementasi
Berdasarkan kebutuhan segera, berikut disarankan urutan prioritas:
1. Panduan Pengguna (User Manual) – karena langsung berpengaruh pada adopsi pengguna.
2. Panduan Instalasi dan Konfigurasi Awal – untuk meminimalkan masalah saat pertama kali instalasi.
3. Panduan Pemecahan Masalah dan FAQ – mengurangi beban support.
4. Panduan Backup dan Pemulihan Data – penting untuk keamanan data sekolah.
5. Dokumen Struktur Database – membantu admin dan integrator memahami data yang dikelola.
6. Panduan Keamanan dan Privasi – meningkatkan kepercayaan pengguna.
7. Panduan Pengembang – jika ada rencana untuk kolaborasi terbuka atau outsourcing pemeliharaan.
8. Dokumen Legal (Privacy Policy, ToS) – kepatuhan dan transparansi hukum.

Setelah dokumen-dokumen ini dibuat dan ditambahkan ke repository (misalnya di folder `docs/` atau sebagai lampiran terpisah dalam release), SmartSPJ akan memiliki dokumentasi yang lengkap dan siap untuk distribusi ke pengguna akhir serta tim teknis.
